OpenClover 5.0.0 Release Notes

2026-06-24

We are thrilled to announce OpenClover 5.0 — the first major release since the 4.x series began, and the result of years of careful work by the community! This release delivers on promises made throughout the 4.5.x cycle and sets the foundation for even bigger things ahead.

What's New

Java

Official Java 17 support — it's here!

Throughout the 4.5.x releases we methodically chased down every missing language construct, edge case, and instrumentation bug for JDK 17. That work is done. We are proud to announce official, production-ready support for Java 17! You can now instrument and compile code at any language level from 8 to 17, and run it under any JDK/JRE from 8 to 17 with full confidence.

Java 21 runtime verified

Planning your JDK upgrade? Good news: OpenClover runs cleanly on Java 21. Note that instrumentation of language features introduced in Java 18–21 is not yet supported — that's what the next cycle is for. But if your build toolchain already runs on JDK 21, you won't be blocked.

Java 1.7 support dropped

To clear the path for Java 21 instrumentation support, we are officially dropping Java 1.7. If your OpenClover configuration specifies 1.7 as the language level, it will be silently treated as 8 and a warning will be printed. Since all OpenClover classes have always required JRE 8+ at runtime, this change affects only a very small number of projects still targeting a JVM that reached end-of-life years ago.

Groovy — finally some love!

Full support for Groovy 3.x through 5.x

Groovy has been waiting for this for a while. We've rewritten OpenClover's Groovy AST transformations from scratch and updated all related libraries to bring solid, tested compatibility with Groovy 3.x, 4.x, and 5.x.

Two exciting new instrumentation capabilities come with this:

  • Lambda functions and method references are now instrumented. You will finally see coverage inside chained calls like stream().map().filter() — no more blind spots in functional-style code.
  • Synthetic default branch instrumentation for switch statements. When a switch has no explicit default case, OpenClover now instruments the implicit one. This lets you catch places where not all cases are handled — a common source of silent NullPointerExceptions.
Heads-up — visibility change: OpenClover has always injected a helper method $CLV_R$() into instrumented classes. Its visibility has changed from private to package-private. If you use a library that scans non-private members via reflection (e.g. a serialization framework), you may need to add an exclusion for this method in that library's configuration.

Groovy 2.x support removed

Groovy 2.x reached end of life years ago, and maintaining compatibility was holding back Groovy 3–5 work. OpenClover 5.0.0 may still work with Groovy 2.x in practice, but we make no guarantees — future bug-fix releases will remove deprecated code without hesitation. Time to upgrade!

Codebase cleanup

Package renaming — goodbye com.atlassian.*

A major version bump was the perfect moment to finish the renaming that started when Atlassian open-sourced Clover. All remaining com.atlassian.* package references have been replaced with proper org.openclover.* prefixes, and each module now has a unique package namespace. For the vast majority of users this is invisible. If you maintain a custom plugin that imports OpenClover's internal Java classes directly, you will need to update those import statements.

Upgrading from OpenClover 4.5.x

Clover for Ant

Download the new Clover-for-Ant ZIP and drop the new clover.jar into your build.

Clover for Eclipse

Download the new update site ZIP and install it in your Eclipse via Help → Install New Software.

Clover for IDEA

Download the new plugin JAR and install it via Settings → Plugins → Install Plugin from Disk.

Clover Maven Plugin

Bump the version to 5.0.0 for both org.openclover:clover-maven-plugin and org.openclover:clover in your pom.xml files. Also ensure your Maven version is at least 3.3.1 (see OC-236).

Jenkins Clover Plugin

Upgrade to the latest 4.16.x plugin release from the Jenkins update centre.
